## 1. 产品概述

### 1.1 产品定位
一款**个人专用的家庭健身训练 App**，支持手机投屏到电视，提供大屏健身体验。

### 1.2 产品愿景
让居家健身更简单、更专业、更有动力。

### 1.3 核心价值
| 价值点 | 说明 |
|--------|------|
| **大屏展示** | 投屏到电视，无需盯着手机小屏 |
| **专业指导** | 标准动作库 + 训练计划 |
| **零成本** | 开源免费，无订阅费用 |
| **隐私保护** | 纯本地使用，数据不出设备 |

### 1.4 使用场景
```
用户下班回家 → 打开电视 → 手机启动 App → 投屏 → 跟随训练 → 完成记录
```

### 4.5 投屏功能 ⭐核心

#### 4.5.1 投屏方案
| 方案 | 技术 | 优先级 |
|------|------|--------|
| 方案 A | Android 系统投屏 (Screen Mirroring) | P0 (现成) |
| 方案 B | 局域网 WebSocket 投屏 | P1 (自定义) |

#### 4.5.2 方案 A：系统投屏（推荐）
**实现方式：**
1. 手机下拉菜单 → 打开"无线投屏"/"Smart View"
2. 选择三星电视
3. 手机屏幕镜像到电视
4. App 自动进入"投屏模式"（横屏、大屏 UI）

**优点：**
- ✅ 无需开发，系统自带
- ✅ 稳定性好，延迟低
- ✅ 三星电视完美支持

**缺点：**
- ❌ 手机不能息屏
- ❌ 手机操作会同步到电视

#### 4.5.3 方案 B：WebSocket 投屏（可选）
**架构：**
```
手机 (Server) ←WebSocket→ 电视浏览器 (Client)
```

**实现步骤：**
1. 手机启动 WebSocket 服务器（端口 8080）
2. 生成连接二维码（含手机 IP）
3. 电视浏览器扫码访问
4. 建立连接，同步训练状态

**消息类型：**
```json
// 训练开始
{"type": "start", "plan": {...}}

// 动作变更
{"type": "exercise", "id": "pushup", "set": 2, "reps": 10}

// 休息
{"type": "rest", "seconds": 60, "next": "squat"}

// 完成
{"type": "complete", "duration": 1200}
```

#### 4.5.4 投屏模式 UI
| 模式 | 手机 | 电视 |
|------|------|------|
| 镜像模式 | 完整 UI | 手机镜像 |
| 双屏模式 | 控制器 | 训练内容 |
| 电视模式 | 隐藏/锁屏 | 完整训练 UI |

### 8.1 技术风险
